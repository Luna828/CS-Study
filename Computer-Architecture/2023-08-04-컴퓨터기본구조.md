# 컴퓨터의 기본 구조

### CPU (Central Processing Unit)

CPU는 CS에서 가장 중요하고 컴퓨터의 `성능`과 `기능`을 결정하는 주요 요소

* 중앙 처리장치 
    - 컴퓨터 시스템에서 `핵심적인` 기능을 수행 (주기억장치)
    - 모든 작업을 `제어`하고 `실행`함
    - 속도가 매우 `빠름`
    - 예) 뇌 
* 기능
1. `명령 해석`` : 메모리부터 가져온 명령어를 해석하고 실행할 수 있는 명령어로 변환
2. `연산`: CPU는 산술 연산 & 논리 연산 수행하여 계산 작업
3. `데이터 이동`: 
    - 메모리로부터 데이터 I/O 가능 (입출력) 
    - 레지스터 간 데이터 이동
4. `제어`: 프로그램 실행 흐름 제어, 분기와 점프 등 명령어 사용하여 다음에 실행할 명령어 위치 결정

CPU는 여러개의 코어로 구성
    - 여러작업 동시 처리 (멀티태스킹)
    - 코어의 수가 多 -> 컴퓨터 성능 향상


### RAM (Random Access Memory)

* 접근하여 읽고 쓰기가 가능한 `주 기억 장치`
* 실행중인 프로그램들과 작업에 필요한 데이터를 `임시보관`
* 휘발성 메모리 (전원이 꺼지면 데이터 소멸)
* 용량이 `크다면`?
    - 동시에 `많은`` 프로그램 실행
    - `대용량`` 데이터 처리 용이
    - 성능 & 반응속도 굳

### HDD (Hard Disk Drive)

* `비휘발성`이며 순차적 접근 가능한 `보조 기억장치`
* 자기 디스크와 이동하는 자기 머리로 이루어져 있으며 데이터를 자기 디스크에서 기록하고 읽어옴

** 자기 디스크란?
- 원판 모양의 회전하는 원반이며 자기적으로 감지되는 미세한 도금이 적용되어져있음 
- 데이터를 0 또는 1 로 저장하기 위해 사용됨 (이진 데이터가 자기적으로 기록되고 읽혀짐)
- 최근에는 SSD라는 플래시 메모를 사용하는 저장 장치가 더 많이 사용됨 (더 빠르고, 낮은 전력 소비 = 성능 굳)

### SSD (Solid State Disk)

* 전자기적으로 데이터를 저장하는 저장 장치
* 전통적 HDD와 다르게 기계적 부품 없이 반도체 메모리 칩 사용하여 데이터 I/O 
* 빠른 속도
* 낮은 전력 소비
* 향상된 내구성
* NAND 플래시 메모리 사용으로 데이터 저장

장점
1. 빠른속도
2. 내구성
3. 에너지 효율성
4. 무음 운영 

위의 4가지 다 기계적인 회전이 없고 부품이 없어 나오는 장점들임!

# 정보의 표현과 저장

컴퓨터의 정보 종류는 2진수 비트들로 표현된 `프로그램 코드` & `데이터` 존재

### 프로그램 코드 

1. 기계어
    - 기계 코드 (machine code)
    - 하드웨어가 이해할 수 있는 언어 -> 2진 비트 구성

2. 어셈블리 언어
    - 고급언어와 기계어 사이의 중간 언어
    - 기계어와 1:1 대응
    - 어셈블리 프로그램을 기계어 프로그램으로 번역하는 소프트웨어

3. 고급언어
    - 영문자와 숫자로 구성되어 사람이 이해하기 쉬운 언어
    - C++, C, Java 등
    - 컴파일러를 이용하여 기계어로 번역

고급언어: 사용자를 위한 언어
기계어는: 컴퓨터가 이해하는 언어 

### 프로그램 코드 & 데이터 기억장치 저장

* 프로그램 코드와 데이터는 지정된 기억장소에 (RAM)에 저장됨
* 단어 단위로 저장
* 낮은 주소에 명령어가 쌓임
* 높은 주소에 데이터가 쌓임

BUT!!
컴퓨터는 프로그램 코드 영역과 데이터 코드 영역을 구분하지 않음

단어란? 
- 정보의 기본단위
- CPU에 의해 한번에 처리될 수 있는 비트들 그룹
- 8bit, 16bit, 32bit, 64bit, 128bit
- 단어의 크기에 따라 컴퓨터가 한번에 처리할 수 있는 데이터 양이 달라짐


# 시스템의 구성

## 시스템 버스

* 컴퓨터의 구성 요소를 연결하고 데이터 전달을 위한 경로
* 주소 버스, 데이터 버스, 제어 버스

### 주소 버스 (Address Bus)

* 메모리 주소 / 포트 번호 전달
* CPU와 메모리는 단방향 데이터 전송이므로 전달흐름은 CPU -> 메모리
* CPU, Memory는 I/O Unit과 양방향으로 데이터 전달이 가능한 버스를 사용

### 데이터 버스 (Data Bus)

컴퓨터에서 기억장치로부터 판독이나 기록을 위해 `데이터` 신호를 각 처리 장치로 `전송`하는 경로

각 장치간 잇는 선은 정보의 흐름을 나타내고 있으므로 버스라고 한다! 

* CPU와 Memory 사이 & CPU와 I/O 장치 사이에서 데이터 전송
* 병렬 버스 구성 (라인별 하나의 비트) 
예시) 32비트 데이터 버스는 한번에 32비트 (4바이트) 데이터 전송 가능
* 각 구성요소(CPU, Memory, I/O Unit)는 양방향으로 데이터 전달이 가능한 버스를 사용 

### 제어 버스 (Control Bus)

CPU가 기억 장치나 입출력 장치에 데이터 전송을 할 때나, 자신의 상태를 다른 장치들에 알리기 위해 사용하는 신호를 전달한다. 

* 제어 신호 전달
* Read & Write 신호 전달
* 각 구성요소(CPU, Memory, I/O Unit)는 양방향으로 데이터 전달이 가능한 버스를 사용 

## CPU와 기억장치의 접속

CPU ----> 메모리
* Load와 store(저장) 명령 수행

1. Load
    - 주소버스: 메모리에서 불러오는 데이터가 저장된 주소값 전달 (CPU -> Memory)
    - 신호 버스: Memory READ 신호 전달 (CPU -> Memory)
    - 데이터 버스: 지정한 메모리에 저장된 값 전달 (Memory -> CPU)

2. Store
    - 메모리에서 해당 데이터를 저장할 주소값 전달(CPU -> Memory)
    - 신호 버스 : Memory Write 신호 전달(CPU -> Memory)
    - 데이터 버스 : 저장할 데이터 전달(CPU -> Memory)

## CPU와 I/O 장치의 접속

CPU <----> I/O Unit

* 입력(Input)과 출력(Output)명령에 의해 수행

1.입력(Input)
    - 주소 버스 : 해당 입출력 장치의 포트 번호 전달(CPU -> I/O Unit)
    - 신호 버스 : `Read` 신호 전달(CPU -> I/O Unit)
    - 데이터 버스 : 해당 입출력 장치는 데이터 전달(I/O Unit -> CPU)

(2) 출력(Output)
    - 주소 버스 : 해당 입출력 장치의 포트 번호 전달(CPU -> I/O Unit)
    - 신호 버스 : `Write` 신호 전달(CPU -> I/O Unit)
    - 데이터 버스 : 입출력 장치에 데이터 전달(CPU -> I/O Unit)

## 전체 시스템의 구성 <위 내용 요약>

위 내용 요약! 

컴퓨터 기본 구조는 아래와 같습니다!

1. CPU

* 모든 작업을 제어하고 실행하는 역할을 하는 `중앙처리 장치`!
* 멀티 코어로 구성되어 작업 동시 처리 -> 성능 향상

기능
1. 명령 해석
2. 연산 처리
3. 데이터 이동
4. 제어

2. RAM

* 프로그램 작업에 필요한 데이터를 저장하는 `주 기억 장치`
* 휘발성 메모리
* 큰 용량일수록 동시에 많은 프로그램, 대용량 데이터 처리 가능 -> 성능 향상

3. HDD

* 비휘발성 보조 기억장치 (순차적 접근 가능)
* 자기 디스크에 데이터를 기록하고 읽어옴

4. SSD 

* 전자기적으로 데이터 저장
* 기계적 부품이 필요없고 메모리 칩을 사용
* 빠른 속도
* 낮은 전력 소비
* 향상된 내구성!

5. 정보의 표현과 저장

* 기본적으로 컴퓨터 정보는 0과 1로 이루어진 이진수 표현법을 따름
* 프로그램 코드와 데이터는 메모리에 저장 
* 각각의 단어 단위로 저장

6. 시스템 버스의 3종류와 접속 관계

시스템 버스란? 
* 컴퓨터 구성요소를 연결하고 데이터 전달을 위한 경로

    1)주소버스 : 메모리 주소 / 포트 번호 전달 
    2)데이터 버스 :  컴퓨터에서 기억장치로부터 판독이나 기록을 위해 `데이터` 신호를 각 처리 장치로 `전송`하는 경로
    3)제어 버스 : CPU가 기억 장치나 입출력 장치에 데이터 전송을 할 때나, 자신의 상태를 다른 장치들에 알리기 위해 사용하는 신호를 전달한다. 

7. CPU와 기억장치의 접속

* CPU와 메모리 간의 데이터 전송은 Load와 Store 명령으로 이루어진다.
* CPU와 기억장치 간의 데이터 전송은 단방향!

8. CPU와 I/O 장치의 접속

* CPU와 입출력 장치 간의 데이터 전송은 입력(Input)과 출력(Output) 명령으로 이루어진다.
* 양방향성 (데이터를 입력하고 출력해야하니까!)

# 컴퓨터 구조의 발전 과정

릴레이 -> 진공관 -> 트랜지스터 -> 반도체 집적회로(IC)

### 릴레이

* 전류의 유무에 따라 회로를 여닫는 장치
* 작동원리: 전자기력을 이용해 기계적으로 스위치 작동
* 한계: 전환 속도가 매우 느리고 기계가 마모됨

### 진공관

* 열 이온 밸브: 최초의 진공관 (밀폐된 유리 전구 안에 두개의 전극을 가둔 것)
* 열 이온 방출: 하나의 전극이 가열되면 전자 방출 (다른 전극이 이 전자를 끌어당김)
* 마모가 적고 1초에 수천번 전환 가능
* 부서지기 쉽고 비싸다는 단점

### 트랜지스터

* 진공관 이후에 속도와 안정성 개선
* 반도체 사용

* 장점:
    - 1초 만번 스위치 작동
    - 안정성
    - 작은 크기 -> 컴퓨터 크기가 작아짐)
    - 집,사무실 컴퓨터 도입 가능
    - 초고속

### 직접회로(IC)

수만개의 트랜지스터들을 하나의 반도체 칩에 집적시킨 전자부품

* 장점:
    - 전기적 통로가 짧아짐 -> 동작 속도 증가
    - 컴퓨터 크기 감소
    - 칩 내부 회로들 간의 상호 연결
    - 전력 소모 
    - 냉각장치 소형화
